# Animal-10: Region-Based Image Classification with SVM & Decision Trees

Classifying 10 animal classes using classical hand-crafted features (color, GLCM, LBP, HOG, edge statistics) with SVM and Decision Tree models — then testing whether classification accuracy can be *improved* by first segmenting each image into superpixels and discarding the regions the model finds unimportant. Built for the **Computational Intelligence** course at Ferdowsi University of Mashhad.

## Overview

The dataset ([Animal-10, Kaggle](https://www.kaggle.com/datasets/viratkothari/animal10)) is class-imbalanced (1,446–4,863 images/class across butterfly, cat, chicken, cow, dog, elephant, horse, sheep, spider, squirrel) and highly non-uniform in image size and aspect ratio. The project runs the same modeling pipeline twice — once on the original images, once on region-pruned images — to measure the effect of automatic "unimportant region removal" on model performance.

| Phase | File | Description |
|---|---|---|
| 1 | [`phase1.ipynb`](phases/phase1.ipynb) | Data exploration & profiling: class balance, image dimensions, color modes, brightness/contrast/color distribution per class. |
| 2 | [`phase2.py`](phases/phase2.py) | Feature extraction: color (RGB+HSV), GLCM, LBP, HOG, edge/contour features → `feature2.csv`. |
| 3 | [`phase3.ipynb`](phases/phase3.ipynb) | Train baseline SVM and Decision Tree (no tuning), then hyperparameter-tune both with `RandomizedSearchCV` (PCA used to cut ~1,800 raw features down for tractable SVM search). |
| 4 | [`phase4.ipynb`](phases/phase4.ipynb) | Evaluation: accuracy/precision/recall/F1 (macro), confusion matrices, per-class error analysis for both models. |
| 5 | [`phase5.ipynb`](phases/phase5.ipynb) | Image segmentation: SLIC superpixels → two-stage clustering (DBSCAN + Agglomerative/Ward) into region masks, region-importance scoring via SVM confidence drop, and masking out the least important regions. |
| 6 | [`phase6_feature.py`](phases/phase6_feature.py), [`phase6.ipynb`](phases/phase6.ipynb) | Re-extract features from the region-pruned images and retrain SVM/Decision Tree on them ("Train2"). |
| 7 | [`phase7.ipynb`](phases/phase7.ipynb) | Re-evaluation on the region-pruned pipeline, and analysis of the resulting accuracy drop. |
| 8 | *(extra credit, in* [`phase5.ipynb`](phases/phase5.ipynb) */* [`phase7.ipynb`](phases/phase7.ipynb)*)* | Apply the same segmentation + region-removal pipeline to the **test** images (not just train) and re-evaluate end to end. |

## Results

**Baseline (phase 3/4, full images, tuned):**

| Model | Test Accuracy | Test Macro-F1 |
|---|---|---|
| SVM (RBF, tuned) | **0.65** | **0.62** |
| Decision Tree (tuned) | 0.35 | 0.31 |

SVM substantially outperforms the Decision Tree, better exploiting the color-similarity structure between classes; the Decision Tree overfits more and struggles especially on `spider` (frequently confused with `dog`, its highest-count class) and `squirrel`.

**After region removal (phase 6/7, pruned images):**

| Model | Test Accuracy | Test Macro-F1 |
|---|---|---|
| SVM (RBF, tuned) | 0.60 | 0.56 |
| Decision Tree (tuned) | 0.29 | 0.22 |

Removing "unimportant" regions *reduced* accuracy by ~5–9 points for both models. The report's analysis (see [`docs/report.pdf`](docs/report.pdf)) attributes this to: the SVM-confidence importance criterion not being perfectly aligned with true class-discriminative regions; useful background/context cues (e.g. grass, webs, antennae) sometimes being pruned away along with actual noise; and the region masking process (flat gray fill) distorting color-statistics and HOG/GLCM features derived near mask boundaries.

Extracted feature CSVs (`feature2.csv`, etc.), trained model `.pkl` files, and the dataset itself are not committed (see `.gitignore`) — they're regenerated by running the phases in order.

## Getting started

```bash
pip install -r requirements.txt
```

Datasets are not committed and must be placed at the repository root as `Dataset/Animals-10/<class>/...` (used by `phase2.py`) and, for the region-removal pipeline, `newDataset_train_all/` and `Dataset/split-animal/` (used by `phase6_feature.py`) — adjust the `DATA_DIR` constants at the top of each script if your local layout differs.

Run in order: `phase1` → `phase2.py` → `phase3` → `phase4` → `phase5` → `phase6_feature.py` → `phase6` → `phase7`.
